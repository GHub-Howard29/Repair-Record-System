# 05_交接紀錄

## 2026-07-12

### 已完成

- 依系統架構文件建立主要 `src` 模組資料夾。
- 新增 `src/agent.ts`，作為後續 agent/開發者快速掌握專案規則與交接文件的入口。
- 建立維修紀錄核心型別：`RepairRecord`、`RepairCharge`、`RepairAttachment`。
- 建立 Phase 1 本機可用雛形：登入佔位、維修紀錄列表、新增/編輯表單、收費摘要、同步規劃提示。
- 建立 Google OAuth 可接線架構：`VITE_GOOGLE_CLIENT_ID`、Google Identity Services 載入、登入狀態保存、本機開發 fallback。
- 落地核心規則：必填欄位驗證、完成後鎖定、同製造號碼不得重複建立未完成案件、保固提示。
- 補上 Phase 2 部分能力：故障零件自動產生零件收費欄位、同製造號碼歷史維修摘要。
- 新增 PWA 基礎：`manifest.webmanifest` 與 service worker。
- 新增工作規範文件：`docs/04_工作規範.md`。

## 2026-07-13

### 已完成

- 補上本機附件管理：新增圖片、最多五張限制、圖片格式檢查、超過 1.5 MB 自動壓縮、預覽、更換、刪除。
- 附件遵守完成後鎖定規則；已完成案件不可新增、更換或刪除附件。
- 附件目前以 `localStorage` 內的 data URL 暫存，狀態標示為 `local`，待後續接 Google Drive 上傳。
- 補上本機搜尋篩選：客戶姓名、製造號碼、回送地點、年月與故障分類。
- 補齊可替換式服務架構：`RepairRecordService`、`AttachmentStorageService`、`ExportService`。
- 建立 Firestore、Google Drive、匯出 placeholder service，後續串接雲端時以 service 實作替換，不需重寫 UI 主流程。
- 建立本機同步佇列 `syncQueue`，維修文字儲存與附件異動會建立待同步任務。
- 同步規劃區已改為依同步佇列顯示待同步與失敗數量。
- 補上同步 processor：離線保留、行動網路附件延後、同步成功清除任務、同步失敗標記 `failed`。
- 補上瀏覽器匯出：單筆維修紀錄建立列印頁面可另存 PDF，全部維修紀錄匯出 CSV 可用 Excel 開啟。
- 依 GitHub Pages 專案頁部署需求設定 Vite `base: '/Repair-Record-System/'`。
- 修正 PWA 相關路徑：manifest、favicon、service worker 註冊會跟隨 Vite base，避免部署到 repo 子路徑後資源失效。
- 新增 `docs/06_外部資源連結設定.md`，整理 Google OAuth、GitHub Pages、Firestore、Google Drive 與部署流程。

- Google 登入入口改用 Google Identity Services 官方按鈕 `renderButton()`，避免 One Tap `prompt()` 出現 `unknown_reason` 略過。

- 更新 manifest、HTML favicon/apple icon、Open Graph / Twitter 縮圖與 service worker 快取清單。

### 尚未完成

- Google OAuth 已有前端接線架構，且已完成正式 `VITE_GOOGLE_CLIENT_ID` 本機驗證。
- Firestore 與 Google Drive 尚未串接，目前資料暫存於 `localStorage`。
- 附件尚未串接 Google Drive，上傳與同步佇列尚未完成。
- 同步佇列目前需手動按「立即同步」，尚未實作背景自動重試。
- PDF / Excel 目前是瀏覽器列印與 CSV 版本，尚未產出正式 PDF 檔與 `.xlsx` 檔。
- 尚未建立自動化測試。

### 下一步建議

1. 到 GitHub repo Settings → Pages，確認 Source 設為 GitHub Actions。
2. 在 GitHub Actions secrets 建立 `VITE_GOOGLE_CLIENT_ID`。
3. push 到 `main`，完成 GitHub Pages 部署驗證。
4. 建立 Firebase project 與 Firestore，替換目前 `localStorage` 存取層。
5. 建立 Google Drive 附件上傳流程與同步佇列，替換目前本機 data URL 暫存。
6. 補上維修紀錄規則測試，先覆蓋必填、鎖定、重複未完成案件與保固判斷。
