# 05_交接紀錄

## 2026-07-12

### 已完成

- 依系統架構文件建立主要 `src` 模組資料夾。
- 新增 `src/agent.ts`，作為後續 agent/開發者快速掌握專案規則與交接文件的入口。
- 建立維修紀錄核心型別：`RepairRecord`、`RepairCharge`、`RepairAttachment`。
- 建立 Phase 1 本機可用雛形：登入佔位、維修紀錄列表、新增/編輯表單、收費摘要、同步規劃提示。
- 建立 Google OAuth 可接線架構：`VITE_GOOGLE_CLIENT_ID`、Google Identity Services 載入、登入狀態保存、本機開發 fallback。
- 落地核心規則：必填欄位驗證、完成後鎖定、同製造號碼不得重複建立未完成案件、保固提示。
- 補上 Phase 2 部分能力：故障零件自動產生零件收費欄位、同製造號碼歷史維修摘要。
- 新增 PWA 基礎：`manifest.webmanifest` 與 service worker。
- 新增工作規範文件：`docs/04_工作規範.md`。

### 尚未完成

- Google OAuth 已有前端接線架構，但尚未提供正式 `VITE_GOOGLE_CLIENT_ID` 驗證實際登入流程。
- Firestore 與 Google Drive 尚未串接，目前資料暫存於 `localStorage`。
- 附件模組尚未實作照片壓縮、預覽、上傳、更換與刪除。
- 搜尋、PDF 匯出、Excel 匯出尚未實作。
- 尚未建立自動化測試。

### 下一步建議

1. 以正式 `VITE_GOOGLE_CLIENT_ID` 驗證 Google OAuth 登入流程。
2. 建立 Firestore repair repository，替換目前 `localStorage` 存取層。
3. 補上維修紀錄規則測試，先覆蓋必填、鎖定、重複未完成案件與保固判斷。
4. 進入附件管理：照片壓縮、預覽、Google Drive 上傳、更換、刪除與完成後鎖定。
