# 07_專案現況

> 最後更新：2026-07-24
>
> 本文件僅保留專案目前可用功能與後續工作，不記錄功能修改過程。

## 已完成功能

### 帳號與資料同步

- 支援 Google OAuth 登入並同步建立 Firebase Authentication session。
- 維修紀錄文字資料可同步至 Firestore；離線資料與同步失敗工作會保留，恢復連線後可再次同步。
- 維修照片可經 Firebase Functions 上傳至 Google Drive，並保存附件 metadata；已有附件可使用本機預覽或 Drive 縮圖備援。
- 已完成案件依 Firestore Rules 限制授權帳號讀寫；系統不提供刪除維修紀錄功能。

### 維修紀錄管理

- 可新增、編輯、查詢維修紀錄；同一製造號碼不可同時存在兩筆未完成案件。
- 收到日期、回送地點、製造號碼為必填；製造號碼固定為 `NIS-` 加 12 碼英數字，會自動轉為大寫。
- 填寫送回日期後，案件與附件會鎖定為唯讀。
- 三個日期欄位支援單一輸入框手動輸入 `YYYY/MM/DD` 與日曆選擇；限制年份 4 碼、月份 2 碼、日期 2 碼，並驗證日期範圍、實際日期與閏年。
- 機器屬性選項為門市客人、網購、展示機；查詢卡片顯示回送地點／客戶名稱／機器屬性。
- 支援故障分類、故障零件複選、零件收費、維修內容、備註、歷史維修摘要與附件照片管理。

### 保固與收費

- 保固期判斷位於故障分類右側，以收到日期與出貨日期相差是否超過 `365 + 30` 天判定保固期內或超過保固期；日期不足或不合法時顯示尚待確認。
- 保固判斷為即時計算，適用於既有紀錄，不會自動改寫備註或收費。
- 檢修測試費、運費與零件費可個別設定；系統自動計算維修總金額。

### 查詢、匯出與介面

- 支援依姓名、製造號碼、回送地點、收到日期區間與故障分類查詢；可切換維修中／已完成狀態。
- 可匯出單筆 PDF；PDF 包含維修資料、保固期判斷、收費內容與附件清單（有附件時）。建議檔名為 `維修報告_YYYYMMDD_客戶名稱.pdf`，日期使用送回日期；手機另存時會保留列印來源以降低 0 KB 檔案風險。
- 可依搜尋結果匯出 Excel，包含「維修紀錄」與「收費明細」工作表。收費明細會排除金額為 0 的項目。
- Excel 使用中文機器屬性與維修總金額欄位；有資料的儲存格具四周細框線。維修內容固定寬度 47、備註固定寬度 27，文字靠左、垂直置中並交由 Excel 自動換行及調整列高。
- 支援桌面工作台與手機窄版功能選單；已建立 PWA manifest、service worker 與應用程式圖示。

### 開發與部署維護

- 根目錄與 `functions/` 各自維護必要的 `node_modules`；Functions 不再依賴根專案，避免產生指向根目錄的 Junction。
- 若暫時不需修改、檢查或部署 Firebase Functions，可移除本機 `functions/node_modules` 釋放空間；重新使用前執行 `npm run functions:install` 即可。
